
# 🗂️ Native Offline Kanban (PWA Ready)

<img width="1133" height="425" alt="image" src="https://github.com/user-attachments/assets/9677ba48-3e08-41b9-9a0c-e7a87e2fd1e4" />

Полностью автономное Kanban-приложение (аналог Trello), работающее без сервера и внешних зависимостей. Весь функционал инкапсулирован в **одном HTML-файле**.

Все данные хранятся локально в браузере. Приложение поддерживает установку как PWA, нативный Drag & Drop и полноценное редактирование карточек с Rich Text.

-----

## 🚀 Основные возможности

  * **Полная автономность** — один файл `index.html`, работающий без Node.js или внешних библиотек.
  * **Offline-first** — поддержка PWA и встроенный Service Worker для работы без сети.
  * **Native Drag & Drop API** — плавное перемещение карточек между колонками.
  * **Rich Text Editor** — форматирование описания (Bold, Italic, списки) через `contenteditable`.
  * **Персонализация** — светлая и тёмная темы, цветовые метки для задач.
  * **Безопасность данных** — автосохранение в `localStorage` и функции экспорта/импорта JSON.
  * **Доступность (A11Y)** — поддержка экранных дикторов (ARIA), управление с клавиатуры и горячие клавиши (Esc, Ctrl+S).

-----

## 📦 Структура проекта

Файл `index.html` имеет монолитную структуру для максимальной переносимости:

  * `<style>` — CSS-переменные (темы), адаптивный layout и анимации.
  * `<main>` — Семантическая разметка (шапка, доска, модальные окна).
  * `<script>` — Логика приложения, обёрнутая в IIFE-модуль `KanbanApp`.

-----

## 🧠 Архитектура JavaScript

Логика приложения разделена на независимые подсистемы внутри защищенного модуля:

| Подсистема | Описание |
| :--- | :--- |
| **Storage** | Работа с `localStorage`, валидация данных при загрузке. |
| **State** | Управление древовидной структурой данных (Колонки -\> Карточки). |
| **Renderer** | Динамическая генерация DOM-элементов "на лету". |
| **Drag & Drop** | Расчет позиций вставки и управление визуальным placeholder. |
| **Editor** | Модальное окно с поддержкой `document.execCommand` для текста. |
| **Service Worker** | Регистрация инлайнового SW через Blob для кэширования. |

## 🛠 Технические подробности

### Хранение данных

Любое изменение (переименование колонки, смена цвета, перемещение) вызывает функцию `persist()`, которая синхронизирует состояние с локальным хранилищем:

```javascript
localStorage.setItem('kanban_v2', JSON.stringify(data));
```

### Логика Drag & Drop

При перемещении карточки используется метод `getDragAfterElement`, который вычисляет ближайший элемент относительно курсора мыши. Это позволяет динамически отрисовывать `drop-placeholder` перед вставкой данных в массив.

### Импорт и Экспорт

Приложение позволяет скачивать текущее состояние в формате `kanban-export-YYYY-MM-DD.json`. При импорте проводится базовая валидация структуры объекта на соответствие схеме "колонки/карточки".

-----

## 🎨 Интерфейс и UI

  * **Колонки:** адаптивная высота, счетчик карточек, inline-редактирование заголовков.
  * **Карточки:** визуальные индикаторы наличия описания, даты создания и цветовые акценты.
  * **Темы:** переключение между `light` и `dark` режимами с сохранением выбора пользователя.
